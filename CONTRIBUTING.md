# Contributing to Turno

[English](./CONTRIBUTING.en.md)

感谢你愿意参与 Turno 的开发与改进。

当前项目仍处于第一阶段 MVP，比较适合围绕以下方向继续推进：

- Web 与 API 的真实联调
- 商品图片上传与展示流程
- 地址管理、评价系统补齐
- 后台管理能力
- 竞价 / 拍卖功能设计与实现

## 开发前准备

请先确认本地环境：

- `Node.js >= 24.0.0`
- `Go >= 1.25.0`
- `MySQL 5.7`
- `Redis 7.x`

启动步骤可参考：`README.md`

## 推荐开发流程

1. Fork 仓库并克隆到本地
2. 创建功能分支，例如：`feat/product-upload`、`fix/order-status`
3. 按 README 完成环境启动
4. 在最小变更范围内完成你的修改
5. 手动验证关键链路
6. 更新相关文档
7. 提交 PR

## 提交前检查

提交前建议至少确认：

- Web 可以正常启动
- API 可以正常启动
- 核心流程没有被破坏：登录、发布商品、收藏、下单、发货、确认收货
- README / API 文档 / 配置说明与实际代码一致

如果你的改动涉及数据库、接口或页面流程，请尽量在 PR 描述中说明：

- 改动目标
- 影响范围
- 是否需要改配置
- 是否需要改数据库
- 如何验证

## 代码与文档建议

- 优先修复根因，不做表层补丁
- 保持改动小而清晰，避免顺手修改无关问题
- 新增功能时，优先补充对应文档
- 如果 README 中的信息已过时，请顺手更新

## Issue 与 PR 建议

### 提交 Bug

建议附上：

- 复现步骤
- 期望结果
- 实际结果
- 报错日志或截图
- 本地环境信息

### 提交功能建议

建议说明：

- 解决什么问题
- 面向哪些用户
- 预期交互或接口变化
- 是否属于 MVP 范围

### 提交 Pull Request

建议一个 PR 只做一件主要事情，便于 Review。

## 当前最需要的贡献方向

- 将 Web 页面从 mock 数据切换到真实 API
- 完善商品上传与图片处理
- 增加订单状态流转的边界校验
- 补充地址管理与评价体系
- 补充后台管理端基础能力

感谢你的参与。
