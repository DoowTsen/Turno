# AI 开发说明

[English](./ai-development.en.md)

本文档记录 Turno 项目中 AI 辅助开发的工具链、协作方式与经验总结。

## 使用的 AI 模型

- **主力模型**：`gpt-5.4`
- **使用方式**：通过 `Codex CLI` 辅助完成需求拆解、代码生成、结构整理、文档撰写与开发协作

## MCP（Model Context Protocol）

当前开发环境已接入以下 MCP 服务：

| MCP 服务 | 用途 |
|----------|------|
| `mcp-mysql-server` | 辅助查看和操作 MySQL 数据 |
| `chrome-devtools` | 辅助页面调试、交互检查与前端联调 |
| `shadcn` | 辅助查询和接入 UI 组件 |
| `pencil` | 辅助进行界面设计与设计稿相关工作 |

## Skill

本项目开发与文档整理中，已使用 / 可配合使用的 Skill 包括：

| Skill | 说明 |
|-------|------|
| `ps-utf8-io` | 避免 Windows PowerShell 中文乱码 |
| `frontend-design` | 适合用于首页、商品页、控制台等界面美化 |
| `collaborating-with-gemini` | 适合做多模型协作评审与原型验证 |

## AI 协作定位

AI 在本项目里更像"开发加速器"，主要帮助完成：

- **需求整理与 MVP 范围收敛** — 将模糊需求转化为可落地的功能清单
- **页面与接口骨架搭建** — 快速生成组件结构与 API 端点
- **数据库与 API 文档初稿整理** — 自动化文档生成与更新
- **README、配置说明和开发流程补全** — 保持文档与代码状态同步

## 经验总结

1. **AI 擅长**：结构化代码生成、样板代码、文档整理、需求拆解
2. **人工把控**：架构决策、业务逻辑细节、边界情况处理、代码质量审查
3. **最佳实践**：用 AI 做初稿 → 人工 Review & 修正 → 迭代优化
